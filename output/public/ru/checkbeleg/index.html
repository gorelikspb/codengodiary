Парсер для текстового формата чека

Что было

После тестирования выяснилось, что нужно сначала сделать парсер для текстового формата, а затем уже работать с OCR. Пользователь предоставил точный формат текста чека с объяснением структуры.

Решение

  • Переделан парсер для работы с текстовым форматом чека
  • Добавлена вкладка для прямого ввода текста (для тестирования)
  • Скрыт функционал с изображением (временно)
  • Парсер использует структуру чека:
  • Начало товаров после строки "EUR"
  • Конец товаров на "------------------------------------------" или "zu zahlen"
  • Товары в строках без отступа
  • Вес/скидка в строках с отступом
  • Общая сумма в "zu zahlen"

Почему такое решение

  • Сначала нужно правильно распарсить текст, затем уже работать с OCR
  • Текстовый ввод позволяет быстро тестировать парсер
  • Структура чека основана на отступах - это ключевая информация

Плюсы

  • ✅ Быстрое тестирование парсера без OCR
  • ✅ Точное понимание структуры чека
  • ✅ Легко отлаживать и улучшать парсер

Минусы

  • ❌ Пока нет работы с изображениями (но это следующий шаг)
  • ❌ Нужно вручную копировать текст из чека

Подводные камни

  • OCR должен правильно распознавать отступы для работы с изображениями
  • Формат чека может отличаться в разных магазинах/регионах

Что сделано

  • Парсер для текстового формата
  • Вкладка для ввода текста
  • Правильное распознавание весовых товаров
  • Правильное распознавание скидок
  • Правильный поиск общей суммы
